# DSE 物理真题知识专题生产标准

## 目的

把 DSE Physics 2021-2025 真题做成长期可扩展的知识专题库。每个独立 HTML 页面不是“做一题”，而是围绕一个知识体系做一篇讲义：先把概念、公式、图像语言、常见误解和考法讲透，再把真题作为验证案例放进来。

目标读者是有一定物理底子、数学能力强、但很久没有系统复习的人。读完后应能恢复模型感，知道这一类题在 DSE 中如何被命题，并能迁移到同类题。

## 当前基准样板

后续每一题开工前，先参考 2024 Q1 和 2024 Q2。

- 2024 Q1：加热曲线不是普通温度-时间图，而是能量账本。先讲斜线段、平台、高度、宽度分别对应什么，再回到原题。
- 2024 Q2：冰水混合不是平均温度题，而是能量账户和相变分叉题。先讲暖水最多放热、0°C 冰先吃 latent heat、最终温度三种分叉，再回到原题。
- 2024 Q3：理想气体的入口不是 vacuum 这道题，而是宏观状态 `p、V、T` 到微观分子数 `N` 的翻译。先讲 mole 与 molecule 两种计数语言、SI 单位纪律、数量级算法，再回到原题。

如果某一题开头已经出现年份、题号、原题关键词或选项判断，就应先按上面三个样板重写为知识体系入口。

## 核心原则

1. 页面开头必须以知识体系为中心，不能以题号、年份、题目陷阱或原题条件开场。
2. 真题英文原文可以出现，但只在“回到考场/原题题面”区域出现一次。
3. 不放原题截图，不贴原卷整页图；若题目依赖图像，必须重画更清楚的教学图。
4. 中文为主，英文只用于专业术语、原题题面、公式名和 DSE 常见表达。
5. 公式首次出现时，必须解释每个字母的含义和单位。
6. 叙述要像讲义或论文式展开，避免 AI 式标签堆叠、口号式总结和一上来罗列信息卡。

## Q1/Q2 提炼出的标准流程

### 0. 开工前定位

先回答三个问题，再写内容：

- 这题背后的知识体系是什么，而不是“这题问什么”。
- 学生真正会蒙在哪里，是概念、图像、单位、相变、数量级，还是符号语言。
- 如果完全不看原题，能否先写出一段独立成立的知识讲解。

标题和开头要写知识判断，而不是题目判断。避免这类开头：

- “这是 2024 Qx 的典型陷阱。”
- “这道题问的是……”
- “题目给了……所以……”

应改成这类开头：

- “加热曲线不是温度随时间变化图，而是一张能量账本。”
- “冰水混合题的第一步，不是算平均温度，而是检查热量是否先被相变吸收。”
- “理想气体分子数估算的入口，不是压强大小的直觉，而是宏观状态到微观计数的翻译。”

### 1. 先写知识主线

前半页只讲知识，不提前代入原题数字，不提前讨论选项。

Q1 的主线：

- 先把坐标轴读清楚：纵轴 temperature，横轴 energy transferred。
- 斜线段代表输入能量造成温度上升。
- 平台代表能量被相变吸收，温度暂时不变。
- 平台高度读 melting point，平台宽度读 latent heat。

Q2 的主线：

- 暖水降到 0°C 只能放出有限热量。
- 0°C ice 不是 0°C water，先要用 latent heat 熔化。
- 若冰未全部熔化，最终温度锁在 0°C。
- 只有冰全部熔化后，才允许继续算高于 0°C 的最终温度。

后续题也照这个结构：先搭建该知识点的完整小系统，再允许真题进场。

### 2. 每个公式都做符号表

公式不要裸露出现。首次出现必须附：

- 字母含义
- 单位
- 适用条件
- 容易误解的地方

例如 Q1：

- `Q = mcΔT`
- `Q`：升温过程中吸收或放出的热量，单位 `J`
- `m`：质量，单位 `kg`
- `c`：比热容，单位 `J kg⁻¹ °C⁻¹` 或 `J kg⁻¹ K⁻¹`
- `ΔT`：温度变化量，单位 `°C` 或 `K`

例如 Q2：

- `Q = mL`
- `Q`：相变过程中吸收或放出的热量，单位 `J`
- `m`：发生相变的质量，单位 `kg`
- `L`：specific latent heat，单位 `J kg⁻¹`

### 3. 有图先重画图，再解释读图动作

图像题不能只说“看斜线段”“看平台”。必须让读者在页面上看得到对应部分。

图像讲解顺序：

- 先说明坐标轴意义。
- 再指出需要看的线段或区域。
- 再解释这个线段或区域对应哪个物理量。
- 最后才说它怎样支持答案。

Q1 的成功点是：页面重画了 heating curve，读者能看到斜线段、水平平台和 P/Q 曲线差异。后续任何图像题都要达到这个标准。

### 4. 真题只在后半段出现一次

知识讲透后，才进入“把这套读法放回考场”。

真题区域标准：

- 使用英文原题题面，标题为 `2024 DSE Qx 原题英文`。
- 原题卡片只出现一次。
- 原题卡片放在应用区，不放在开头知识区。
- 题面之后紧接中文解释：这句英文题面真正给了哪些物理条件，应该触发哪套知识系统。

结构上，`questionBrief` 可以写在 narrative 的最后一个 section 中，但生成页面时只能由应用区统一渲染；不要在知识段落中重复显示题面。

### 5. 回题时不能换成“答案讲解模式”

回到真题后，仍然沿用前面建立的知识路线。

Q1 的做法：

- 不是直接说 C 正确。
- 而是先问 option C 要比较什么量。
- 再回到 solid heating section。
- 再说明斜率在 `T-Q` 图里代表 `1/(mc)`。
- 最后推出 P 的 specific heat capacity 较大。

Q2 的做法：

- 不是直接算最终温度。
- 而是先算 warm water 最多放热。
- 再算 melt all ice 需要多少热量。
- 比较后发现冰没熔完。
- 最后才落到 0°C 和选项 B。

后续题也一样：不能一进真题就“代公式算答案”，必须让读者看见知识体系如何支配解题路径。

### 6. 选项分析要解释诱因

错选项不能只写“错误”。要说明为什么学生会选它，以及错在知识体系的哪一层。

标准写法：

- A 错，是因为把平台高度误读成另一物理量。
- C 错，是因为跳过了 latent heat 门槛。
- D 错，是因为把 cm³ 换算方向弄错。

这会让页面像研究一类题，而不是只公布答案。

### 7. 结尾必须给迁移规则

结尾要收束成下次遇到同类题时的第一反应。

Q1：

- 斜线段读 specific heat capacity。
- 平台高度读 melting point。
- 平台宽度读 latent heat。

Q2：

- 看到 0°C ice，先问热量够不够把所有冰熔化。
- 只要冰还剩，最终温度就是 0°C。

## `cases.ts` 落地结构

每题至少按以下顺序组织：

1. `title`：知识判断，不是题号摘要。
2. `subtitle`：说明这套知识如何帮助读题。
3. `knowledgeSystem.bigIdea`：独立知识主题，不能一上来写“这道题”。
4. `knowledgeSystem.narrative.heading`：知识体系标题。
5. `narrative.lead`：从物理现象或模型开始，不提前展示原题。
6. `narrative.sections[0..2]`：知识讲解、公式、图像、误解。
7. `narrative.sections[last].questionBrief`：英文原题卡片数据。
8. `bridgeToQuestion`：进入考场的桥，但不能在页面前半段抢跑。
9. `thinkingPath` / `solution`：按知识路线推回答案。
10. `optionAnalysis`：逐项解释诱因。
11. `highScoreReflex` / `variationDrill`：迁移规则和变式。

如果不是图像题，在 `narrative.reasoningHeading` 里改掉默认标题，例如：

- 图像题：`从图像慢慢推到答案`
- 数量级题：`从单位和数量级慢慢推到答案`
- 能量题：`从能量账户慢慢推到答案`

## 开工前检查清单

每做新题前，必须先检查：

- 我是否已经用一句话写出“这题背后的知识体系”。
- 页面第一屏是否没有题号、年份、选项和原题条件抢戏。
- 前三节是否即使不看原题也能作为独立讲义成立。
- 公式是否都有字母含义和单位。
- 若依赖图像，是否已经重画教学图。
- 原题英文是否只会出现一次，并且只在应用区。
- 解析是否先走知识路线，再落到答案。
- 错选项是否解释了误选原因。

## 验收命令

内容改完后运行：

```bash
npm run generate:physics-cases
npm run build
```

然后检查对应页面：

```bash
rg -n "2024 DSE Qx 原题英文|question-brief exam-question-card" public/physics/dse-2024/p1/qxx/index.html
```

验收标准：

- `question-brief exam-question-card` 只出现一次。
- `2024 DSE Qx 原题英文` 只出现一次。
- 页面开头没有直接讲题目。
- build 通过；Vite chunk warning 可以记录，但不阻塞内容验收。

## 后续执行约定

从下一题开始，先读本文件，再改内容。若用户指出“还是先讲题目”，优先回到本标准的第 0-1 步重写开头，而不是只在原有题解前面补几段文字。
